Servidor y Hardware

Local Host

Para cumplir con el requisito de acceder a la página mediante la dirección 127.0.0.1 (o localhost), se utilizó el módulo integrado de servidor HTTP de Python. Este método es ligero, rápido y no requiere la instalación de software adicional como XAMPP o WAMP. Se abrió el Explorador de Archivos y se copió la dirección completa de la carpeta que contenía el proyecto (ejemplo: C:\Users\TuUsuario\Documents\Proyecto web).

Se abrió el Símbolo del sistema (cmd) mediante la combinación de teclas Windows + R.

Se ejecutó el comando cd seguido de la ruta copiada (ejemplo: cd C:\Users\TuUsuario\Documents\Proyecto web).

Una vez que la Terminal estuvo posicionada en la carpeta del proyecto, se ejecutó el siguiente comando para iniciar el servidor HTTP en el puerto 8000: python -m http.server 8000

Y ya estaría finalizada.

Tras ejecutar el comando, el servidor se activó. Finalmente, se accedió a la página web a través de cualquiera de las siguientes direcciones en el navegador, cumpliendo con el requisito de usar la dirección local: http://127.0.0.1:8000

UEFI

Prenderemos la computadora o la más común es reiniciar la computadora ya iniciada.

Cuando aparezca el logo de la marca de la computadora (DELL) presionamos repetidas veces la tecla F2 o también dejarla presionada, aparece una barra de carga en el inferior que indicara que estamos accediendo a la BIOS y un mensaje en la parte superior derecha “Preparing to enter BIOS setup” otro indicativo de que estamos entrando a la BIOS.

Una vez hecho esto donde podemos observar una versión mas avanzada de la BIOS conocido como UEFI, el cual contiene un distintivo color blanco y celeste además de las múltiples opciones para modificar ciertas configuraciones para poder habilitar el uso de correcto de aplicaciones o deshabilitar cosas que no usemos o creamos no tan relevantes.

-Secure Boot: También conocido como arranque seguro, es un modo implementado en el firmware UEFI de Windows desde Windows 8, este impide la ejecución de cualquier software no firmado o certificado en el arranque del sistema. Esto no ayuda a evitar la carga de malware o aplicaciones que no sean confiables cuando uno inicia el ordenador, pero también afecta sistemas operativos como GNU/Linux. Fernández, Y. (2019, 18 abril). Ver fuente

-Post Behavior (Power-On Self Test): También llamado prueba automática de encendido es una operación que inicia una computadora encenderla, pero antes del inicio del sistema operativo. El Firmware (UEFI) realiza esta operación ejecutando una secuencia de pruebas de diagnóstico para determinar si el hardware esencial de la computadora funciona correctamente.

Este es esencial para la resolución de problemas en el sistema ya que el arranque completo solo se puede realizar si el software funciona correctamente, en el caso de que no la BIOS/UEFI mostrara un mensaje de error. Sheldon, R. (2022, 2 agosto). Ver fuente

- Power Managment: Consiste en una opción que determina que tan agresivo se administrara energía a la CPU y se colocara en modo turbo. Power | UEFI_XEON_3RD | Lenovo Docs. (s. f.). Ver fuente

AMD B650 Chipset Este chipset de gama media, está diseñado para usuarios que buscan un equilibrio entre rendimiento y precio, ofreciendo soporte para procesadores AMD Ryzen de serie 7000, 8000 y 9000. Ahora yendo a la memoria B650 esta es compatible con módulos DDR5, soportando frecuencias a base de la placa base. Este modelo se ha descontinuado y un enfoque al B850, pero eso no quiere decir que ya no estén disponibles en el mercado.
